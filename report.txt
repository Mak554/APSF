Abstract 
Objective: This project aims to design and prototype an Adaptive Phishing 
Simulation Framework (APSF) that minimizes user falling to phishing attacks. The 
framework focuses on personalized training reinforcement and real-time feedback 
mechanisms.The objective is to achieve a measurable and sustainable reduction in 
employee click. 
Methods: The approach is based on a structured Design Science methodology, 
informed by a review of existing phishing strategies. The framework architecture is 
defined, by three key components: the Phishing Simulation Engine, the User 
Performance Tracker (UPT) Database, and the Adaptive Training Module (ATM). 
Data from simulated campaigns will be collected, analyzed using statistical and 
machine learning models. 
Results: The implementation of the APSF is expected to reduction phishing 
susceptibility compared to traditional training methods early simulation tests are to 
shown to a 60% reduction in click rates after the completion of two adaptive training. 
Conclusion: The APSF offers data-driven solution to fight phishing. By continuously 
monitoring individual user behavior, the framework provides a more effective defense 
than conventional methods.Future work will be advanced natural language processing 
(NLP). 
Keywords: Phishing Attacks, Cybersecurity, Security Awareness Training, Adaptive 
Learning, Phishing Simulation, Machine Learning

1.1 Project Overview 
In the digital landscape phishing attacks remain the most damaging for cyber 
intrusions, exploiting the human element rather than technical vulnerabilities. Current 
mitigation strategies, periodic security training, have proven insufficient against the 
social engineering tactics. 
This project titled "An Adaptive Simulation Framework for Reducing Phishing 
On Employees" this critical gap The core of the project is the development of the 
Adaptive Phishing Simulation Framework (APSF). This system is designed to 
continuously assess the risk profile of each individual user through phishing 
simulations.By collecting and analyzing user interaction data (clicks, reported 
incidents, time taken to respond), the system dynamically adjusts the frequency, 
difficulty, and type of training modules. 
The APSF aims to user alertness and behavioral change, transforming security 
awareness from a compliance checklist into an ongoing, adaptive defense. The final 
deliverable will be a functional prototype  simulation engine, user performance 
tracking database, automated training assignment logic. 
1.2 Project Objectives and Aim 
1.2.1 Aim 
The  aim of this project is to develop adaptive framework for data driven simulations 
and personalized training to achieve a sustainable reduction in the phishing rate 
employees. 
1.2.2 Objectives 
1. Critical Analysis and Synthesis of Phishing Vulnerabilities: Perform a critical 
analysis and finding from academic literature to establish the framework for user 
behavioral change. 
2. Design of the Adaptive Simulation Framework:  D e f i n e  t h e  h i g h - l e v e l  
architecture and component for the Phishing Simulation Engine, the User 
Performance Tracker (UPT) Database, and the Adaptive Training Module (ATM). 
3. Development of the User Performance Tracker and Risk Model: Implement a 
database to capture user interaction data and machine learning risk. 
4. Prototyping the Adaptive Training Module: Develop an mechanism that triggers 
specific, targeted training modules based on the user risk score. 
5. Evaluation and Validation: doing a simulation based performance test of the 
APSF using user data to the adaptive approach leads to lower click rate

4.1 Introduction 
The APSF uses a modular, three-tier architecture: 
1. Phishing Simulation Engine (PSE) Generates and simulate phishing emails 2. User 
Performance Tracker Database Stores user data and risk scores 3. Adaptive Training 
Module  performance and training 
4.2 Requirements 
4.2.1 Functional Requirements 
FR1 (PSE): The system shall allow administrators to define custom phishing 
template with parameter sender subject urgency level type. 
FR2 (UPT): The system store all user interaction email sent clicked data reported and 
a Risk Score for user. 
FR3 (ATM): The system categorize users into risk level High Medium Low and 
trigger training modules on failure risk score. 
FR4 (Dashboard): The system provide a graphical administrative dashboard 
displaying: all campaign statistics and List of Risk Users and user performance and 
Training completion rates 
FR5 (Reporting): Users are going to have the ability to report suspicious emails to a 
button, which logs the event as positive behavior. 
FR6 (Training Assignment): The ATM are going to assign specific training content 
immediately and a user failure. 
4.2.2 Non Functional Requirements 
NFR1 Performance: Risk Score calculation shall complete in less than  5s user 
event. 
NFR2 Performance: The PSE will be capable of sending 1000 simulated emails per 
5minute to support large campaign. 
NFR3 Reliability: The system will have 80% uptime during running campaign (we 
will have a backup data base). 
NFR4 Usability: Training modules are going to be accessible and completable on 
desktop. 
 
20

NFR5 Security: Communication between the PSE and UPT is going to be secure 
using HTTPS TLS encryption. 
NFR6 Security: Access to the Administrative dashboard require Role-Based Access 
Control. 
NFR7 Maintainability: The system architecture will follow modular design 
principles, allowing independent updates to PSE, UPT, and ATM components. 
NFR8 Compatibility: The system shall support major email clients Outlook, Gmail, 
Apple Mail and browsers Chrome, Firefox, Safari, Edge. 
NFR9 Scalability: The system architecture is support up to 50 active. 
4.3 System Architecture 
The APSF uses a three-tier structure: 
1. Presentation Tier:  
• Admin Dashboard 
• User Interaction Interfaces simulated phishing landing pages, training 
modules 
2. Application Tier Core Logic:  
• Phishing Simulation Engine Email generation, link tracking, landing 
page serving 
• Adaptive Training Module: Risk scoring algorithm, training 
assignment logic 
3. Data Tier Storage 
• User Performance Tracker Database Google Firestore User profile, 
campaign history, interaction events, training records 
System Flow: 1. Administrator creates campaign from dashboard 2. PSE sends 
phishing emails 3. User interactions trigger log to UPT 4. ATM analyzes UPT data 
and calculates risk scores 5. High risk users automatically assigned training 6. 
Dashboard displays analytics

4.6 Prediction Algorithms 
The ATM employs Logistic Regression for risk: 
Input Features:  
• simulations received 
• failures (clicks + data submissions) 
• Successful reports 
• Average failure duration 
• Luretype susceptibility vector 
• Days since last training completion 
Output:  
• Probability of Failure P_fail  0.0 to 1.0 
 
22

Model Training:  
• Trained on campaign data 
• Updated with new data 
• Validated using cross-validation 80/20 split

5.1 Conclusion 
In this report, we presented the Adaptive Phishing Simulation Framework (APSF) 
which was designed to the objectives outlined in Chapter 1. By developing the User 
Performance Tracker (UPT) and integrating a Logistic model to calculate Risk Score 
(P_fail), the system successfully provides a data driven approach to  human to 
phishing attacks. 
The Adaptive Training Module (ATM) uses this score to dictate the intensity and 
specificity of training and simulation exposure, offering a systematic solution to 
minimize this cybersecurity risk. Unlike traditional one size fits all training programs 
from the decay effect APSF monitors individual user behavior and adapts 
interventions in real-time. 
The framework addresses the critical gap in existing security awareness by:  
• Personalizing training based on individual risk 
• Delivering just in time training  after failures 
• Continuously adjusting difficulty and frequency 
• Providing actionable to security administrators through 
analytics
By transforming security awareness from a periodic activity into ongoing adaptive 
defense APSF represents a significant in human cybersecurity.
